# Football Bot — Plan

## Context
Групповой Telegram-бот для 3–4 человек (whitelist). Игроки собирают команды из карточек футболистов, участвуют в еженедельных турнирах с авто-симуляцией матчей. Бот работает в двух режимах: групповой чат (анонсы, симуляция, таблица побед) и личные сообщения (настройка состава/схемы).

---

## Tech Stack

- **Python 3.12** + **aiogram 3.x** (async Telegram bot)
- **SQLite** (через `aiosqlite` + `SQLAlchemy async`) — достаточно для 3–4 игроков
- **APScheduler** — еженедельные крон-задачи (раздача паков, запуск турнира)
- **API-Football** (rapidapi.com) — бесплатный тир 100 req/day для первоначального сбора базы игроков
- **LLM (Claude API)** — переводит события матча в текст, публикует матч несколькими сообщениями
- **Статичная БД игроков** — собирается один раз скриптом из API, хранится в SQLite, дополняется вручную (сборная России)

## Данные игроков

### Источник
API-Football (v3) через RapidAPI:
- `GET /players?league=XXX&season=2024` — игроки с рейтингом
- Рейтинг = `statistics[0].games.rating` (среднее за сезон) → масштабируем в 65–99
- **Игроков с рейтингом ниже 65 не добавляем в базу**

Сборные: загружаем через API (`GET /players?team=XXX&season=2024`).
Сборная России — **вручную CSV** (игроки РПЛ с ручными рейтингами).

## Паки

### Стартовый пак (только при первом /start)
Выдаётся один раз при первом запуске бота. Состав:
- По **1 игроку на каждую позицию** согласно схеме 4-4-2 (11 карточек)
- **+3 запасных** на случайные позиции
- Итого: **14 карточек**
- Вероятность рейтингов чуть хуже обычного пака, но **гарантирован минимум 1 игрок 85+**

| Рейтинг | Вероятность |
|---------|-------------|
| 65–74 | 65% |
| 75–84 | 27% |
| 85+ | 8% (но 1 гарантирован) |

---

### Один вид пака (5 карточек)
Вероятность выпадения по рейтингу:
- 65–74: 55%
- 75–84: 35%
- 85+: 10%

Выдаётся **автоматически в четверг 10:00** каждому игроку из whitelist.

Также админ может **вручную выдать крутой пак** командой `/givepak @user [gold]` — тогда вес 85+ повышается.

## Турнир

### Расписание
- **Среда 20:00** — анонс "Сегодня турнир! Настрой состав в ЛС до 21:00"
- **Среда 21:00** — сбор составов, запуск симуляции
- **Четверг 10:00** — раздача паков + анонс результатов

### Формат
Round-robin (каждый с каждым). Результаты публикуются в чат.

---

## Симуляция матча

### Алгоритм (умный + рандомный)
1. Для каждой команды: `team_strength = weighted_avg(player_ratings)` где веса зависят от позиции в схеме
2. `expected_goals = poisson(λ)` где λ нелинейно зависит от разницы силы команд + рандом-фактор
3. Для каждого гола — определяется **кто забил** по вероятности позиции:

| Позиция | Вероятность гола | Вероятность ассиста |
|---------|-----------------|---------------------|
| Striker (ST/CF) | 35% | 10% |
| Attacking Mid (CAM/SS) | 20% | 25% |
| Winger (LW/RW) | 18% | 20% |
| Central Mid (CM) | 10% | 20% |
| Defensive Mid (CDM) | 5% | 10% |
| Full back (LB/RB) | 7% | 12% |
| Centre back (CB) | 4% | 2% |
| Goalkeeper (GK) | 1% | 1% |

4. LLM (Claude) получает события матча (кто забил, на какой минуте, позиции) и генерирует текстовый комментарий
5. Матч публикуется **несколькими сообщениями** (по голу / важным моментам)

### Схема состава
Поддерживаемые схемы: 4-4-2, 4-3-3, 3-5-2, 5-3-2.
Каждый слот привязан к позиции. Игрока можно поставить на любой слот, но за несоответствие позиции — штраф к рейтингу:

Из API-Football у игрока есть список его позиций (основная + дополнительные). Штраф:
- Своя/дополнительная позиция → **0**
- Чужая позиция в своей зоне (нап→AM или AM→CM) → **-5**
- Чужая зона (нап→защита) → **-10**
- Вратарь поля или полевой в ворота → **-30**

### Дефолтный состав
Если игрок не настроил состав — берутся топ-11 карточек по рейтингу, схема 4-4-2, автоматически распределяются по позициям.

### Скиллы LLM-агента
Используется механизм скиллов (будет показан отдельно). При реализации — смотреть туда.

## Статистика (команды в чате и ЛС)

### В групповом чате
| Команда | Описание |
|---------|----------|
| `/standings` | Турнирная таблица (побед/ничьих/поражений/участий) |
| `/top` | Лучшие бомбардиры за всё время |
| `/topweek` | Лучшие бомбардиры текущего турнира |
| `/topassists` | Лучшие ассистенты за всё время |
| `/myteam @user` | Текущий состав (или свой если без @) |

### В личных сообщениях
| Команда | Описание |
|---------|----------|
| `/squad` | Настроить состав (inline-клавиатура по слотам) |
| `/formation` | Выбрать схему |
| `/mycards` | Вся коллекция |
| `/mystats` | Мои бомбардиры/ассистенты за всё время |
| `/transfer @user` | Предложить обмен карточкой |
| `/transfers` | Активные предложения |

## Трансферы

- **3 трансфера в неделю** на игрока (счётчик сбрасывается в четверг вместе с паками)
- Механика: обмен карточками
  1. `/transfer @username` → выбрать свою карточку → указать какую хочешь в ответ
  2. Другой получает уведомление в ЛС, принимает/отклоняет inline-кнопками
  3. При принятии — карточки меняются в `user_cards`, счётчик -1 у инициатора

## Whitelist

Таблица `whitelist`. Управление через `/adduser <user_id>` (только ADMIN_ID из `.env`).

## Реализация по шагам

1. Структура проекта + requirements.txt + .env.example
2. DB модели (SQLAlchemy async)
3. Whitelist middleware
4. Скрипт сбора игроков `fetch_players.py` (лиги + сборные + Россия CSV)
5. Пак-система (`packs.py`) + анонс в чат
6. Симуляция матча (`simulation.py`) с вероятностями позиций
7. LLM-комментатор (`llm_commentator.py`) — скиллы подключить когда покажут
8. Турнир round-robin + публикация
9. Статистика голов/ассистов + команды `/top`, `/standings`, `/mystats`
10. Трансферы flow (offer → accept/decline)
11. Handler'ы: group, private, admin
12. Scheduler (среда + четверг)
13. main.py — точка входа
